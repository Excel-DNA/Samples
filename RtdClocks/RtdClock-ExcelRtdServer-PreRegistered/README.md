# RtdClock-ExcelRtdServer-PreRegistered

This sample shows the pre-registered COM server path for an Excel-DNA `ExcelRtdServer`.

The simpler sibling sample, `RtdClock-ExcelRtdServer`, uses:

```csharp
XlCall.RTD(...)
```

That is the Excel-DNA on-demand registration path. This sample instead uses:

```csharp
XlCall.Excel(XlCall.xlfRtd, RtdClockServer.ServerProgId, null, "")
```

That is equivalent to entering this formula in a worksheet:

```excel
=RTD("RtdClock.ClockServer",,"")
```

For that formula to work, Excel must be able to resolve `RtdClock.ClockServer` through COM and load the XLL as the in-process COM server.

## Convert On-Demand RTD to Pre-Registered RTD

Start from a working on-demand RTD server. That proves your RTD implementation is basically sound. If pre-registration does not work, the missing step is usually COM identity, `.dna` metadata, registration bitness, or registry writes.

### 1. Give the RTD server stable COM identity

The RTD server class needs to be COM-visible and should have both `[Guid]` and `[ProgId]`:

```csharp
[ComVisible(true)]
[Guid("296F3518-6374-4F71-B35F-6938346C6076")]
[ProgId(RtdClockServer.ServerProgId)]
public class RtdClockServer : ExcelRtdServer
{
    public const string ServerProgId = "RtdClock.ClockServer";
}
```

The `[ProgId]` is the name used by `=RTD(...)`. The `[Guid]` is the COM CLSID written under the registry.

### 2. Mark the add-in as a COM server

There are two project shapes.

If the project has an explicit `.dna` file in the project folder, edit that file directly:

```xml
<ExternalLibrary Path="RtdClock-ExcelRtdServer-PreRegistered.dll"
                 ExplicitExports="false"
                 ComServer="true"
                 LoadFromBytes="true"
                 Pack="true" />
```

If the `.dna` file is generated by the SDK-style build, there is no source `.dna` file to edit. Put this property in the `.csproj` instead:

```xml
<PropertyGroup>
  <ExcelAddInComServer>true</ExcelAddInComServer>
</PropertyGroup>
```

After building, check the generated `.dna` under the output folder. You should see:

```xml
<ExternalLibrary ComServer="true" Path="RtdClock-ExcelRtdServer-PreRegistered.dll" ... />
```

If that attribute is missing, Excel-DNA will not include the assembly's COM classes in the persistent COM registration list.

### 3. Register the XLL as the COM server

The normal persistent registration path is `regsvr32`.

For the 64-bit XLL:

```bat
%windir%\System32\regsvr32.exe RtdClock-ExcelRtdServer-PreRegistered-AddIn64.xll
%windir%\System32\regsvr32.exe /u RtdClock-ExcelRtdServer-PreRegistered-AddIn64.xll
```

For the 32-bit XLL, use `%windir%\SysWOW64\regsvr32.exe`.

This sample also exposes two Excel commands under the Add-ins tab:

* `Register RTD COM Server`
* `Unregister RTD COM Server`

Those commands call Excel-DNA's internal equivalent:

```csharp
ComServer.DllRegisterServer();
ComServer.DllUnregisterServer();
```

Use the commands for quick local experiments. Use `regsvr32` or an installer when you want the server registered before Excel loads the add-in.

### 4. Check the registry

After registration, check that the ProgId maps to the class GUID:

```bat
reg query HKCR\RtdClock.ClockServer\CLSID
```

Then check that the CLSID maps to the XLL:

```bat
reg query HKCR\CLSID\{296F3518-6374-4F71-B35F-6938346C6076}\InProcServer32
```

The default value should be the full path to the `.xll`, and `ThreadingModel` should be `Both`.

Use the registry view that matches Excel. From a 64-bit shell, `reg.exe` checks the 64-bit view. For 32-bit Excel on 64-bit Windows, use:

```bat
%windir%\SysWOW64\reg.exe query HKCR\RtdClock.ClockServer\CLSID
```

`HKCR` is a merged view. Excel-DNA may write to machine classes or per-user classes depending on permissions, so checking `HKCR` with the correct bitness is usually the simplest test.

### 5. Test from Excel

After registration, test the direct worksheet formula:

```excel
=RTD("RtdClock.ClockServer",,"")
```

You can also test the wrapper UDF:

```excel
=dnaRtdClock_ExcelRtdServer_PreRegistered()
```

If the on-demand sample works but this formula does not, check these items in order:

* the RTD server class has both `[ComVisible(true)]` and a stable `[Guid]`
* the ProgId in the formula exactly matches the `[ProgId]`
* the generated or explicit `.dna` has `ComServer="true"`
* the XLL bitness matches the `regsvr32.exe` bitness
* `HKCR\RtdClock.ClockServer\CLSID` exists
* `HKCR\CLSID\{296F3518-6374-4F71-B35F-6938346C6076}\InProcServer32` points to the XLL
